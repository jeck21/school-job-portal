3. Create `scripts/seed-zip-coordinates.ts`:
   - Reads pa-zip-codes.csv using Node fs
   - Parses CSV rows
   - Uses Supabase client to upsert into zip_coordinates table in batches of 100

   - Requires SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY env vars (same as scraper)

4. Create migration `00005_geocode_existing.sql`:
   - UPDATE jobs SET location = extensions.st_point(zc.longitude, zc.latitude)::extensions.geography FROM zip_coordinates zc WHERE jobs.zip_code = zc.zip_code AND jobs.location IS NULL
   - Same UPDATE for schools table
   - NOTE: This runs AFTER zip_coordinates is seeded. Include a comment explaining the dependency.

IMPORTANT: PostGIS uses ST_Point(longitude, latitude) -- longitude FIRST (x, y order). Add comments at every ST_Point call.

2. Create migration `00006_search_jobs_rpc.sql` with the search_jobs RPC function. Follow the exact pattern from 04-RESEARCH.md Pattern 1:
   - Parameters: search_term TEXT, school_types TEXT[], grade_bands TEXT[], subject_areas TEXT[], cert_types TEXT[], salary_only BOOLEAN, zip_lat FLOAT, zip_lng FLOAT, radius_miles FLOAT, include_unspecified BOOLEAN, include_remote BOOLEAN, result_offset INT, result_limit INT
   - Returns: id, title, description, location_raw, city, school_type, grade_band, subject_area, salary_mentioned, certifications, first_seen_at, last_verified_at, url, zip_code, school_name, district_name, total_count
   - Include description, grade_band, subject_area, salary_mentioned, certifications, last_verified_at, zip_code in the return (the existing getJobs doesn't return these but filter UI needs them)
   - Keyword search: ILIKE on title, location_raw, and school name (NOT description, per user decision)
   - School type: = ANY(school_types) with include_unspecified fallback for NULL
   - Grade band: && array overlap with include_unspecified for NULL/empty arrays
   - Subject area: && array overlap with include_unspecified for NULL/empty arrays
   - Certification: && array overlap with include_unspecified for NULL/empty arrays
   - Salary: filter salary_mentioned = true when salary_only is true
   - Radius: extensions.st_dwithin with miles * 1609.34 conversion to meters. longitude FIRST in st_point.
   - Remote: when include_remote AND radius active, also include jobs with school_type = 'cyber'
   - SET search_path = '' for security (qualify all with public. and extensions.)
   - ORDER BY first_seen_at DESC, OFFSET/LIMIT for pagination
