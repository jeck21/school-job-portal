<objective>
Fix the verified-only filter to use server-side filtering in the search_jobs RPC instead of client-side post-filtering on paginated results.

Purpose: The current implementation filters the 25-row page slice client-side, causing wrong counts and broken pagination when >25 verified jobs exist. Moving the filter server-side lets COUNT(*) OVER() and OFFSET/LIMIT work correctly.

Output: Updated RPC function, fixed server action, fixed client component, and passing tests.
</objective>
