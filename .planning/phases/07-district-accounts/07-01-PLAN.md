Create migration 00008_district_accounts.sql with:

1. ALTER districts table -- add columns:
   - `slug TEXT UNIQUE` (auto-generated from name)
   - `verified BOOLEAN NOT NULL DEFAULT false`
   - `verified_at TIMESTAMPTZ`

2. CREATE TABLE district_accounts:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE` with UNIQUE constraint
   - `district_id UUID NOT NULL REFERENCES districts(id)`
   - `email TEXT NOT NULL`
   - `email_domain TEXT NOT NULL`
   - `is_verified BOOLEAN NOT NULL DEFAULT false`
   - `verified_at TIMESTAMPTZ`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - Apply the existing `update_updated_at` trigger

3. CREATE TABLE verified_domains:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `domain TEXT NOT NULL UNIQUE`
   - `added_by TEXT` (value: 'auto' for .k12.pa.us pattern, or admin email for manual)
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`

4. ALTER jobs table -- add columns:
   - `claimed_by_district_id UUID REFERENCES districts(id)`
   - `claimed_at TIMESTAMPTZ`
   - `delisted_at TIMESTAMPTZ`
   - `is_manual BOOLEAN NOT NULL DEFAULT false`

5. Create indexes:
   - `idx_jobs_claimed_district ON jobs(claimed_by_district_id) WHERE claimed_by_district_id IS NOT NULL`
   - `idx_districts_slug ON districts(slug) WHERE slug IS NOT NULL`
   - `idx_districts_verified ON districts(verified) WHERE verified = true`

6. Enable RLS on district_accounts and verified_domains:
   - district_accounts: SELECT for own user_id (`auth.uid() = user_id`)
   - districts: SELECT for all (public can read for profiles)
   - jobs: UPDATE for own claimed jobs (claimed_by_district_id matches user's district via district_accounts)
   - jobs: INSERT for manual jobs (is_manual = true AND claimed_by_district_id matches user's district)
   - verified_domains: SELECT for all authenticated users (to check whitelist)

7. Update search_jobs RPC to also filter out delisted jobs: add `AND j.delisted_at IS NULL` to the WHERE clause. Use CREATE OR REPLACE FUNCTION to update it in place.

8. Also add `claimed_by_district_id UUID` to the search_jobs RETURNS TABLE so the UI can show verified badges. Map it from `j.claimed_by_district_id` in the SELECT.

1. Create `src/lib/supabase/middleware.ts` -- the updateSession helper:
   - Use createServerClient from @supabase/ssr
   - Accept NextRequest, return NextResponse
   - Use getUser() (NOT getSession()) for session validation
   - Protect `/for-schools/dashboard` routes: redirect unauthenticated users to `/for-schools/login`
   - Follow the exact pattern from 07-RESEARCH.md Pattern 1

2. Create `src/middleware.ts` -- root middleware:
   - Import and call updateSession
   - Matcher: `/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)`

3. Create `src/lib/domain-validation.ts`:
   - Export `validateDomain(email: string): { autoVerified: boolean; domain: string }`
   - `.k12.pa.us` domains return autoVerified: true
   - All others return autoVerified: false (whitelist check happens at DB level in the confirmation callback)
   - Export `generateSlug(name: string): string` -- simple: lowercase, replace non-alphanumeric with hyphens, trim leading/trailing hyphens

4. Create `src/lib/actions/auth-actions.ts` -- server actions following report-job.ts pattern:
   - `signup(formData: FormData)`: Extract email + password + districtName. Call supabase.auth.signUp with emailRedirectTo pointing to `${process.env.NEXT_PUBLIC_SITE_URL}/auth/confirm`. On success, redirect to `/for-schools/signup?check-email=true`.
   - `login(formData: FormData)`: Extract email + password. Call supabase.auth.signInWithPassword. On success, redirect to `/for-schools/dashboard`.
   - `logout()`: Call supabase.auth.signOut. Redirect to `/for-schools`.
   - All actions use `await createClient()` from server.ts. Return errors as `{ error: string }` rather than throwing (for form error display).

5. Create `src/app/auth/confirm/route.ts` -- email confirmation callback:
   - GET handler: extract token_hash and type from searchParams
   - Call supabase.auth.verifyOtp({ token_hash, type })
   - On success: get user email, call validateDomain, then:
     a. If autoVerified or domain is in verified_domains table: create district_accounts row with is_verified=true, find or create district by name (using districtName from user metadata), set district.verified=true, generate slug
     b. If not auto-verified: create district_accounts row with is_verified=false
   - Redirect to `/for-schools/dashboard`
   - On error: redirect to `/for-schools/login?error=confirmation`
   - Use service role client (createClient from supabase-admin or create inline) for inserting district_accounts since RLS restricts inserts

6. Replace `src/app/for-schools/page.tsx` (currently coming-soon):
   - Landing page for districts: brief value prop ("Manage your school's job listings"), two CTAs: "Log In" and "Sign Up" linking to respective pages
   - Forest & Ember theme styling

7. Create `src/app/for-schools/login/page.tsx`:
   - Form with email + password fields using shadcn Input/Label/Button
   - Form action calls login server action
   - Show error state from URL params (error=confirmation, error=invalid)
   - Link to signup page

8. Create `src/app/for-schools/signup/page.tsx`:
   - Form with email + password + district name fields
   - Store districtName in user metadata via options.data in signUp
   - Form action calls signup server action
   - After signup, show "Check your email" message (when ?check-email=true)
   - Link to login page

Use the warm amber CTA button styling (`bg-cta text-cta-foreground` classes) for primary actions.
