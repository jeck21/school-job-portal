   - Join with schools table to get district_name
   - Use `compareTwoStrings` from string-similarity to fuzzy-match districtName against schools.district_name

   - Return array of `{ jobId, title, schoolName, score }` sorted by score descending
   - Use admin/service client if needed (since RLS may restrict reads for new accounts)

2. Create `src/lib/queries/get-district-jobs.ts`:
   - `"use server"` directive
   - Export `getDistrictJobs(districtId: string, filter: 'active' | 'delisted')`:
     - active: claimed_by_district_id = districtId AND is_active = true AND delisted_at IS NULL
     - delisted: claimed_by_district_id = districtId AND delisted_at IS NOT NULL
   - Return jobs with id, title, school name, first_seen_at, is_manual, delisted_at, url
   - Order by first_seen_at DESC

3. Create `src/lib/actions/claim-actions.ts`:
   - `"use server"` directive
   - Export `claimJobs(jobIds: string[], districtId: string)`:
     - Verify the authenticated user belongs to this district (check district_accounts)
     - Update jobs SET claimed_by_district_id = districtId, claimed_at = now() WHERE id IN jobIds
     - Return { success: true, claimed: count }

4. Create `src/lib/actions/listing-actions.ts`:
   - `"use server"` directive
   - Export `delistJob(jobId: string)`:
     - Verify user owns this job's district
     - UPDATE jobs SET delisted_at = now(), is_active = false WHERE id = jobId
   - Export `relistJob(jobId: string)`:
     - Verify user owns this job's district
     - UPDATE jobs SET delisted_at = null, is_active = true WHERE id = jobId
   - Export `createManualJob(formData: FormData)`:
     - Extract: title, description, school (free text or select from district's schools), grade_band, subject_area, salary_raw, url (application URL), expires_at (deadline)
     - Get user's district_id from district_accounts
     - Find or create source row for 'manual' source (name: 'District Portal', slug: 'manual')
     - Insert into jobs with is_manual = true, claimed_by_district_id = districtId, claimed_at = now(), source_id from manual source
     - Return { success: true, jobId }
   - Export `updateManualJob(jobId: string, formData: FormData)`:
     - Verify user owns this job's district AND job.is_manual = true
     - If is_manual is false, return { error: "Cannot edit scraped listings" }
     - Update job fields from formData
   - All actions use `await createClient()` and check auth via getUser()

5. Modify `scripts/scrapers/lib/ingest-pipeline.ts` -- delist suppression:
   - In the processBatch function, before the job upsert (around line 249-273):
   - After building jobRecord, check if an existing job with the same source_id + external_id has delisted_at set
   - Query: `SELECT delisted_at FROM jobs WHERE source_id = $sourceId AND external_id = $externalId`
   - If delisted_at IS NOT NULL, skip setting is_active: true (keep current state) -- set is_active to undefined or omit it from the upsert record so the DB default/current value is preserved
   - Log when a delisted job is encountered: `[ingest] Skipping is_active update for delisted job: ${job.externalId}`

6. Modify `scripts/scrapers/lib/ingest-pipeline.ts` -- auto-claim for verified districts (per locked decision: "new scraped listings matching a verified district are auto-claimed and badged"):
   - After a successful job upsert (new job inserted, not an update to existing), check if the job should be auto-claimed
   - Query: look up the job's school via schools table to get district_name, then check if any row in district_accounts has is_verified = true for a district whose name fuzzy-matches (use `compareTwoStrings` from string-similarity, threshold 0.8) OR exact-matches the school's district_name
   - Simpler alternative (preferred): join schools -> districts where districts.verified = true AND schools.district_name matches districts.name. Use a direct SQL match first (exact), falling back to the existing fuzzy match only if needed
   - If a verified district match is found and the job's claimed_by_district_id IS NULL: UPDATE jobs SET claimed_by_district_id = matched_district_id, claimed_at = now() WHERE id = new_job_id
   - Log: `[ingest] Auto-claimed job ${job.externalId} for verified district: ${districtName}`
   - This runs only for newly inserted jobs (not updates to existing jobs that are already claimed)
