    - computeDedupScore("Math Teacher", "Springfield SD", "Math Teacher", "Springfield School District") >= 0.8 (match)
    - computeDedupScore("Math Teacher", "Springfield SD", "Science Teacher", "Springfield SD") < 0.8 (different title)
    - computeDedupScore("Math Teacher", "Springfield SD", "Math Teacher", "Harrisburg SD") < 0.8 (different school)
    - normalizeForDedup("  Math Teacher (K-4)  ") returns "math teacher k4"
    - Borderline scores (0.7-0.85) are identified for review logging
    - Score weighting: title 0.6, school 0.4
  </behavior>
  <action>
    1. Create `scripts/scrapers/lib/job-dedup.ts`:
       - `normalizeForDedup(text: string): string` -- lowercase, strip non-alphanumeric except spaces, collapse whitespace
       - `computeDedupScore(titleA, schoolA, titleB, schoolB): number` -- weighted Dice coefficient (title 0.6, school 0.4)
       - `findDuplicate(supabase, newJob: {title, schoolName}, excludeSourceId): Promise<{jobId, score} | null>` -- queries active jobs from other sources, returns best match above 0.7 threshold
       - Constants: DEDUP_MATCH_THRESHOLD=0.8, DEDUP_REVIEW_LOW=0.7, DEDUP_REVIEW_HIGH=0.85
       - Use `compareTwoStrings` from existing `string-similarity` package

    2. Create `scripts/scrapers/lib/ingest-pipeline.ts`:
       - Extract the shared pattern from `ingestPareap` into a generic `runIngestion(adapter, sourceConfig)` function
       - `sourceConfig` param: `{name, slug, baseUrl, scraperType}`
       - Pipeline: ensureSource -> createScrapeLog -> adapter.scrape() -> getExistingExternalIds -> processBatch (with dedup) -> updateScrapeLog
       - In processBatch: before upserting a new job, call `findDuplicate`. If score >= 0.8, link to existing job via job_sources instead of creating new job. If score 0.7-0.85, log as borderline for review. If score < 0.7 or no match, create new job.
       - When dedup match found: use the most complete version (longest description) as canonical per user decision. Update existing job record if new version has longer description.
       - Batch size: 25 (consistent with PAREAP)

    3. Create `tests/scrapers/job-dedup.test.ts`:
       - Test normalizeForDedup with various inputs
       - Test computeDedupScore with exact matches, partial matches, no matches
       - Test borderline detection (0.7-0.85 range)
       - Test title vs school weighting

    4. Create `tests/scrapers/fixtures/paeducator-job.json` with a sample PAeducator API response for use in Task 2 tests.
