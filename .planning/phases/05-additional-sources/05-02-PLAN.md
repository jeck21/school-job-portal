  <files>

  <behavior>
    - parseSchoolSpringListing(html) extracts date, title, school, location, and detail URL from each table row
    - Rows without 4+ cellData TDs are skipped (header rows, empty rows)
    - Pagination continues until page returns no results or 50-page safety cap
    - Empty HTML returns empty array
    - Job IDs extracted from detail page href (e.g., "jobId=12345" or path segment)
  </behavior>
  <action>
    1. Create `tests/scrapers/fixtures/schoolspring-listing.html`:
       - Sample HTML matching the verified SchoolSpring structure: `<tr>` rows with `<TD class="cellData">` cells containing [date, title+link, school, location]
       - Include 3-4 sample job rows plus a header row

    2. Create `scripts/scrapers/adapters/schoolspring/types.ts`:
       - `SchoolSpringListingRow`: `{date: string, title: string, detailUrl: string, school: string, location: string}`

    3. Create `scripts/scrapers/adapters/schoolspring/parser.ts`:
       - `parseSchoolSpringListing(html: string): SchoolSpringListingRow[]`
       - Use cheerio to parse `<tr>` rows, find `td.cellData` cells
       - Extract: cells[0]=date, cells[1]=title+href, cells[2]=school, cells[3]=location
       - Skip rows with fewer than 4 cellData cells

    4. Create `scripts/scrapers/adapters/schoolspring/index.ts`:
       - `SchoolSpringAdapter implements SourceAdapter` with `sourceSlug = "schoolspring"`
       - `scrape()` method:
         a. Loop pages 0 through 49 (safety cap)
         b. For each page, POST to `https://employer.schoolspring.com/find/pennsylvania_teaching_jobs_in_pennsylvania.cfm` with form data: `pageNumber={N}&ssPageNumber={N}` (Content-Type: application/x-www-form-urlencoded)
         c. Parse HTML response with `parseSchoolSpringListing`
         d. If page returns 0 rows, stop pagination
         e. Map each row to ScrapedJob:
            - externalId: extract job ID from detailUrl (parse URL for jobId param or path)
            - title: row.title
            - url: row.detailUrl (SchoolSpring detail URL -- this is the best available link since SchoolSpring doesn't expose employer direct URLs in listing page)
            - locationRaw: row.location
            - city: parse city from "City, Pennsylvania" format
            - state: "PA"
            - schoolName: row.school
            - postedDate: parse row.date (e.g., "Mar 11" -> ISO date string for current year)
         f. Add 1s delay between page requests
         g. Log progress every 5 pages
       - Use `fetchWithRetry` for HTTP requests (do NOT set NODE_TLS_REJECT_UNAUTHORIZED -- SchoolSpring has valid SSL)

    5. Create `scripts/scrapers/adapters/schoolspring/ingest.ts`:
       - `ingestSchoolSpring()` calls `runIngestion(new SchoolSpringAdapter(), {name: "SchoolSpring", slug: "schoolspring", baseUrl: "https://employer.schoolspring.com", scraperType: "cheerio"})`

    6. Create `tests/scrapers/schoolspring.test.ts`:
       - Test parseSchoolSpringListing with fixture HTML (row count, field extraction)
       - Test header row skipping
       - Test empty HTML handling
       - Test date parsing ("Mar 11" -> date string)
       - Test externalId extraction from detail URL

    7. Add `scrape:schoolspring` to package.json scripts
  </action>

    <automated>npx vitest run tests/scrapers/schoolspring.test.ts -x --reporter=verbose</automated>
  </verify>
  <done>SchoolSpring parser correctly extracts jobs from HTML fixture. Adapter handles pagination. Unit tests pass.</done>
</task>
