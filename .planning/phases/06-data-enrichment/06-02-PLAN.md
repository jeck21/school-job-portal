  <name>Task 2: Create GitHub Actions weekly freshness workflow</name>
  <files>.github/workflows/freshness.yml</files>
  <action>
    Create `.github/workflows/freshness.yml` for weekly freshness checks:

    ```yaml
    name: Freshness Check

    on:
      schedule:
        - cron: '0 8 * * 0'   # Sundays at 8 AM UTC (3-4 AM Eastern)
      workflow_dispatch: {}     # Manual trigger with no inputs

    jobs:
      freshness:
        runs-on: ubuntu-latest
        timeout-minutes: 60     # Longer timeout -- checks all active jobs
        env:
          SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'
          - run: npm ci
          - name: Run freshness check
            run: npx tsx scripts/scrapers/freshness/run.ts
    ```

    Key design decisions:
    - Sunday 8 AM UTC chosen to avoid overlap with daily scrape crons (6/10/14/18 UTC on weekdays)
    - 60-minute timeout because checking all active job URLs takes longer than scraping
    - `ANTHROPIC_API_KEY` is optional -- if missing, AI calls are skipped gracefully (ai-analyzer.ts handles this)
    - `workflow_dispatch` with no inputs allows manual trigger from GitHub Actions UI
    - No `NODE_TLS_REJECT_UNAUTHORIZED=0` at workflow level -- the check-freshness.ts script handles TLS per-domain
  </action>
